**Known infra bug surfaced by issue 24**: compose `frontend` service can't reach `api` because `vite.config.ts` proxies `/api` to `http://localhost:8000` (the container itself, not the host). Workaround: run `npm run dev` on the host. Tracked as issue 25.

**Known agent registry gotcha** (surfaced during PR 31): the Task tool's `subagent_type` registry is loaded once at session start and does **not** refresh when a new project-local agent file lands mid-session. The `smoke-tester` agent (added by issue 24, merged) was unreachable in the same session that added it; e2e specs had to be run inline by the main session. A session restart picks the agent up. Worth a one-liner in `CLAUDE.md` "Known gotchas" — fold into the next dx PR.

**Convention worth landing**: write bare issue references like "issue 23" in `followup.md` rather than the `#23` form. GitHub's keyword scanner can interpret literal `#NN` in any subsequent merge commit body as a closing reference, which is how issue 23 got auto-closed without a fix landing. Both reviewers on PR 28 and PR 30 flagged this independently. In PR bodies the `#N` form is correct (it's needed to trigger intentional close-on-merge). Worth a short addition to `CLAUDE.md` "Conventions specific to this repo".
