Autofocused name input; Enter submits; ESC cancels; inline validation for blank name; inline error for server failures; form stays open on error with focus retained so user can fix + retry. On success: `resetForm()` returns card to idle with cleared input; new column is already visible via the store's local-append path. Store chose **option A** (add `createColumn` action to existing `useBoardsStore`) over option B (new `useColumnsStore`) — single column action so far (create) makes a dedicated store premature; filed no issue, will re-evaluate when #83 (rename) + #84 (delete) + #85 (reorder) UIs ship. Local-append (not re-fetch) chosen: cheaper, avoids spinner-flash, bounded state-vs-server divergence — backend's append-only position math (`MAX(position) + 1000`, PR #147) guarantees the new column lands in the same slot the next GET would return. Cross-board guard on the local append (only mutate `currentBoard.columns` if `currentBoard.id === boardId`) defends against fast user-navigation races — covered by a dedicated vitest case. Errors rethrow so form owns its own error UI; store does NOT touch `currentBoardError` (reserved for detail-load failures). Four new vitest cases on `boards.spec.ts` (now 17): POST + append + return value, cross-board guard, rethrow + untouched columns on error, in-flight `creatingColumn=true`.
