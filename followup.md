**Known auth bug surfaced by #4**: `auth_service.authenticate` short-circuits when `get_by_username` returns `None`, skipping argon2 verify. Local measurements show wrong-password ≈ 29 ms vs unknown-user ≈ 4.5 ms — a ~6× timing side-channel that lets an attacker probe usernames. The strict timing-parity assertion is deliberately deferred from `test_login_response_parity_wrong_vs_unknown` until the service runs a dummy argon2 hash on the user-not-found branch. Tracked as #23.

**Known infra bug surfaced by #24**: the compose `frontend` service can't reach the `api` service because `vite.config.ts` proxies `/api` to `http://localhost:8000` (which is the container itself, not the host). Workaround documented in `frontend/tests/e2e/README.md` and the smoke-tester agent — run `npm run dev` on the host while compose runs only the backend services. Tracked as #25.
