## Задача 2 — Агентный режим (аналог Дня 2)

**Постановка:** «Найди все места, где хардкожен размер поля (20×20), вынеси в
константу `BOARD_SIZE` в `utils/constants.ts`, прогони тесты.»

Это задача на кросс-файловое рассуждение: нужно пройтись `grep`-ом, править
несколько файлов, запускать `bun test`.

### Результаты

| Модель | С первого раза? | Качество | Скорость |
|---|---|---|---|
| Qwen3.6-35B-A3B MLX @ 4bit через Codex CLI | 🟢 Да | Нашёл 4 места, корректно импортировал константу, тесты прошли | ~4 мин (включая инференцию + запуск `bun test`) |
| Qwen3.5-9B MLX @ 4bit через Codex CLI | 🟡 Частично | Нашёл 3 из 4, четвёртое (в тестовом файле) пропустил. `bun test` упал — разобрался только после подсказки | ~3 мин + ~4 мин на ручную доводку |
| Qwopus 3.5 9B v3 @ 4bit через Codex CLI | ❌ Не завершил | После второго `read_file` снова зациклился, Codex отбил по таймауту | — |

**Ключевое наблюдение для агентного режима:**
локальная 9B-модель проваливается именно на сценариях, где нужно держать в
голове состояние нескольких файлов одновременно. 35B MoE держит контекст
заметно лучше — вероятно, за счёт большего общего числа параметров (экспертов),
даже если в каждый момент активны только 3B.

## Зафиксированные метрики

| Метрика | 35B MoE 4bit | 9B dense 4bit |
|---|---|---|
| Prefill скорость (ввод 8k промпт) | ~180–220 tok/s | ~90–120 tok/s |
| Decode скорость (генерация) | ~22–28 tok/s | ~35–45 tok/s |
| Время первого токена при 30k контексте | 4–6 сек | 8–12 сек |
| Стабильность на 4+ шагах агента | Высокая | Средняя |
| Случаи OOM за тестовый прогон | 0 | 2 (64k ctx, длинный диалог) |

> Prefill на MoE быстрее, несмотря на бо́льшую общую модель, потому что
> forward-pass проходит только через активных экспертов (3B). Decode
> ограничен bandwidth → dense 9B тут немного выигрывает, но разница меньше
> критичной.

## Что не получилось вообще локально

- **Длинный рефакторинг через весь репозиторий** (изменить API хука и
  починить всех потребителей в 10+ файлах) — и 35B начал терять детали к
  5–6 файлу. С облаком (Claude) это шло без осечек.
- **Работа с редкими/свежими API** — локальные Qwen-модели обучены по
  данным с отсечкой, новые фичи Vite 6 / React 19 помнят хуже облака.
- **Поиск по интернету** — MCP не заработал (см. [01-setup.md](01-setup.md)).
  Локально любой вопрос типа «а как сейчас правильно в Vitest 2.x» — мимо.
